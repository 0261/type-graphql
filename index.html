Interoperable

Although TypeGraphQL is data-layer library agnostic, it integrates well with other decorator-based libraries, like TypeORM, sequelize-typescript or Typegoose.

This allows you to define both the GraphQL type and the entity in a single class - no need to jump between multiple files to add or rename some properties.

Validation

Forget about manual inputs and arguments validation! No need to create custom scalars to limit the length of a string or a int range.

Just use decorators from class-validator library and declare the requirements of the inputs. It couldn't be easier!

Easy testability

Dependency injection support and decorators abstraction gives a great separation of business logic from the underlying transport layer.

Thanks to this, you can easily mock the dependencies to prevent side effects and unit test your resolvers like a simple services that only takes some inputs and returns result.
